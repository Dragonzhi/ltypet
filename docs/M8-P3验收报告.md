# M8 P3 验收报告：编辑器动作接入生产桌宠

> 实施日期：2026-07-20  
> 对应计划：[`M8-P3实施计划.md`](./M8-P3实施计划.md)  
> 当前结论：代码与核心真实窗口路径通过；进入 P4 前建议由用户完成第 7 节人工矩阵

## 1. 结论

P3 已完成生产接入。单击、键盘和 Mock Agent 仍只提交 `motion.play("wave")`，动作依次经过 `BehaviorScheduler → PetActionExecutor → SvgCharacterRenderer → SvgMotionPlayer`，不再通过 React `data-action` 或 CSS 关键帧旁路播放。

正式 `wave` 的唯一生产来源是 `src/assets/character/xiaoluobao/motions.v1.json`。旧 `pet-wave`、前后层级 CSS 动画、前景右手 clone 和 10 秒 `animationend` 安全计时器已经删除。

## 2. 生产资产闭环

正式资产集中在：

```text
src/assets/character/xiaoluobao/
  artwork.svg
  rig.v1.json
  motions.v1.json
```

- `artwork.svg` 来自已通过 P0-P2 验证的 `小洛宝.glax.svg`。
- rig 生成脚本直接读取并写回上述目录；共享核心测试也读取同一正式 rig，不再维护第二份小洛宝正式 fixture。
- 加载器依次验证 rig schema/语义、素材文件名、canonical SHA-256、viewBox、33 个 Part 的唯一 source binding，以及 motions schema/语义和必需 `wave`。
- 生产 `wave` 由已验收的 P0 导出基线升级为 `id: "wave"`、`loop: "none"`，并补入旧招手观感对应的抬臂、摆动和 `body ↔ front` 离散层级帧。以后可用编辑器 canonical 导出的同名 Clip 原位替换。

## 3. 运行时实现

- 根 Vite 与 TypeScript 通过公共入口 `@ltypet/character-motion` 解析共享核心；根依赖显式加入 `ajv@8.20.0`。
- `loadCharacterMotionBundle` 只负责版本化资产和诊断。
- `playbackMath` 以绝对帧处理 elapsed/fps/speed/repeat，并收集跨帧、跨循环事件。
- `SvgRuntimeRig` 为 Part 建立 authored wrapper，只写矩阵、opacity 和 render slot；完成、中断与释放会恢复默认姿态和 DOM 顺序。
- `SvgMotionPlayer` 同时只保留一个 RAF session，完成、中断、AbortSignal、reduced motion、reset 和 dispose 共用一次性 settle 路径。
- `SvgCharacterRenderer` 的 motion capabilities 来自校验后的动作库；loading、未知动作与损坏资产分别给出稳定错误。
- `breathing`、`blinking`、`pointer-follow`、`hair-physics`、`ear-twitch` 均有独立抑制入口；动作结束后恢复。
- 基础表情和动作事件表情分层，`blink`、`mouthOpen`、`mouthClose` 可受控执行，`sfx/custom` 只产生诊断。

## 6. 已知边界

- 当前正式 `wave` 是从 P0 导出基线与既有 CSS 招手轨迹合并得到的生产启动 Clip，不代表最终美术定稿；可在 Animation Studio 中继续润色后替换同名 Clip。
- `.glax.svg` 与原始 Inkscape SVG 的视觉差异仍属于 P0 已记录的素材边界，本阶段没有重绘素材。
- Animation Studio 自身仍有约 1.52 MB 的 svgcanvas 主 chunk；该代码未进入桌宠生产 bundle。

## 7. 用户人工复核清单

进入 P4 前建议在真实桌面完成：

- [ ] 单击、Enter/Space、DebugConsole 分别播放一次 `wave`。
- [ ] 动作中再次单击，确认前一动作中断且新动作正常开始。
- [ ] 动作中拖动窗口，确认动作中断，拖动、马尾惯性和松手恢复正常。
- [ ] 角色外透明区域仍可点穿，角色轮廓内仍可点击、拖动和右键。
- [ ] 右键菜单、托盘显示/退出和 `Ctrl+Shift+Q` 不退化。
- [ ] Windows 100%/150%/200% 缩放及副屏负坐标下 pivot、点击命中和拖动正常。
- [ ] 开启“减少动态效果”后动作快速完成，不阻塞后续调度。
- [ ] 修改 motions 后 HMR 不产生重复 wrapper、残留 RAF 或 pivot 漂移。
- [ ] 连续播放/中断 100 次后无残留 inline pose、递增 listener、RAF 或 timeout。
