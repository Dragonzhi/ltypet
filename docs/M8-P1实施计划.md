## 1. 结论与启动边界

P0 的代码范围已经完成，svgcanvas 方向可以保留，允许开始 P1。但 P0 当前是
“有条件通过”，所以 P1 必须从 `P1-0 收口门` 开始；在收口门通过前，不得宣布
`rig.json` / `motions.json` schemaVersion 1 已冻结。

P1 的核心目标不是继续堆编辑器 UI，而是得到一个完全不依赖 React、DOM、
svgcanvas 和 Tauri 的纯 TypeScript 动画核心，使以下两端最终使用同一语义：

```text
Animation Studio ─┐
                  ├─ @ltypet/character-motion ─ rig/motion v1
SvgCharacterRenderer ─┘
```

P1 完成后应能在纯单元测试中回答：

1. 一份 artwork 如何绑定为稳定语义 Part。
2. pivot 如何从素材标记换算到 Part 局部坐标。
3. 任意时间点每个 Part 的 authored transform 是什么。
4. logical parent 如何继承，render slot 如何独立决定绘制顺序。
5. 无效 rig/motion 为什么被拒绝。
6. 同一数据序列化后是否稳定、可复现、可生成指纹。

### 2.2 必须在 schema 冻结前处理

## 4. P1-0 收口门

P1-0 可以修改 P0 工具和测试，但不能先写正式 schema 再回头猜坐标语义。

### 4.1 Pivot 局部坐标换算

在浏览器工作副本中取得：

```text
partWorldMatrix  = partElement.getCTM()
pivotWorldPoint  = pivotElement.getCTM() × pivotElementLocalCenter
pivotInPartLocal = inverse(partWorldMatrix) × pivotWorldPoint
```

`pivotElementLocalCenter`：

- `circle` / `ellipse`：`(cx, cy)`。
- `rect`：`(x + width/2, y + height/2)`，仅作为兼容输入。
- 其他标签：P1-0 拒绝并给出明确诊断，不使用 `getBBox()` 静默猜测。

要求：

- `getCTM()` 缺失或 matrix 不可逆时阻止导出 rig。
- 换算结果必须是有限数值。
- 同一素材重复载入结果在容差内一致。
- 100%、150%、200% 页面缩放不改变 SVG 用户坐标结果。
- `pivot_arm_right` 的真实结果替换实验 `{0,0}`。

### 4.2 预览矩阵组合

每帧都从 bind matrix 重新组合，禁止累乘上一帧：

```text
authored = Translate(x, y)
         × Translate(pivot.x, pivot.y)
         × Rotate(rotation)
         × Scale(scaleX, scaleY)
         × Translate(-pivot.x, -pivot.y)

previewLocal = bindMatrix × authored
```

写回工作副本时统一使用 `matrix(a b c d e f)`，避免 SVG transform list 顺序产生歧义。

停止、暂停归零、切换 Part、重新导入、HMR 和 dispose 后恢复原始 transform
属性的“是否存在”与原值。

### 4.3 真实浏览器验收

在 Edge 或 Chrome 打开 `http://localhost:1430`，完成：

- 100%、150%、200% 缩放选择 `arm_right`，选择框与图形一致。
- 显示换算后的 pivot，并确认右手围绕肩部连接点旋转。
- 连续循环 10 次无累计漂移。
- 暂停任意帧、继续、停止、重新播放结果一致。
- 导出 v1 前置样例、刷新、重新导入后结果一致。
- HMR 三次后只有一个舞台和一套监听。

结果、截图和实际浏览器版本写回 P0 报告附录。

### 4.4 导入安全路径统一

只保留一个 `inspectSvgForImport()` 入口：

1. 解析 XML。
2. 完整遍历所有元素的任意 `on*` 属性。
3. 拒绝 `script`、`foreignObject`。
4. 拒绝 `http:`、`https:`、`//`、`javascript:` 外部引用。
5. 检查 CSS `url(...)`；P1-0 默认只允许同文档 `url(#id)`。
6. 检查重复/缺失 ID、重复 label、孤立 pivot 和不可逆 matrix。
7. 只有零 error 才允许调用 `setSvgString()`。

诊断 UI 与 adapter 必须消费同一个结果，不能一边报告 error、一边仍载入危险 SVG。

### 4.5 素材来源和指纹

删除工具目录中的两份素材副本，默认样例直接从仓库 `src/assets/` 以 `?raw`
方式导入。若 Vite 开发服务器需要读取工具根目录之外的文件，只允许仓库根目录，
不开放任意本机路径。

文本指纹规范：

```text
canonicalText = UTF-8 decode
              -> CRLF/CR 统一为 LF
              -> Unicode NFC
              -> 保留其他字符与末尾换行语义
fingerprint   = sha256(canonicalText)
```

实验 JSON、rig artwork 指纹和同步测试都使用同一函数。

素材决策：

1. 同时用原始 SVG 和 `.glax.svg` 完成 svgcanvas 导入与 pivot 验证。
2. 如果原始 SVG 已稳定通过，P1 优先直接使用原始 SVG，取消派生依赖。
3. 如果原始 SVG 失败而 `.glax.svg` 通过，才实现并测试可复现的规范化脚本。
4. 不接受无法复现、只能人工生成的派生文件成为唯一生产 artwork。

### 4.6 P0 临时文件清理

- 删除 `tools/motion-editor/debug_jsdom.mjs`。
- P0 实验 parser 可以保留为迁移输入，但不能继续被编辑器主路径使用。

## 7. Rig v1 数据模型

第一版建议结构：

```ts
interface CharacterRigV1 {
  schemaVersion: 1;
  rigId: string;
  artwork: {
    source: string;
    fingerprint: `sha256:${string}`;
    viewBox: [minX: number, minY: number, width: number, height: number];
  };
  renderSlots: string[];
  parts: RigPartV1[];
}

interface RigPartV1 {
  id: string;
  sourceBinding: {
    kind: "inkscapeLabel" | "elementId" | "dataPart";
    value: string;
  };
  logicalParentId: string | null;
  defaultRenderSlot: string;
  pivot: { x: number; y: number; space: "partLocal" };
  bindMatrix: [a: number, b: number, c: number, d: number, e: number, f: number];
  tags?: string[];
}
```

约束：

- `rigId`、Part ID、slot 和 tag 使用 `[a-z][a-z0-9_-]*`。
- Part ID 唯一，`sourceBinding` 组合唯一。
- `renderSlots` 唯一且数组顺序就是从后到前的绘制顺序，不能排序。
- `logicalParentId` 必须存在或为 null，整图无环。
- `defaultRenderSlot` 必须在 renderSlots 中。
- pivot 和矩阵全部有限。
- bind matrix 行列式绝对值必须大于统一 epsilon。
- viewBox width/height 必须大于 0。
- `sourceBinding` 不保存 svgcanvas Element 或临时 resolved DOM ID。
- 当前素材默认以 `inkscapeLabel` 绑定；换装可使用不同的底层 DOM ID。

## 8. Motions v1 数据模型

```ts
interface MotionLibraryV1 {
  schemaVersion: 1;
  rigId: string;
  clips: MotionClipV1[];
}

interface MotionClipV1 {
  id: string;
  fps: number;
  durationFrames: number;
  loop: "none" | "repeat";
  tracks: PartTrackV1[];
  events: MotionEventV1[];
  suppressProceduralChannels?: ProceduralChannel[];
}

interface PartTrackV1 {
  partId: string;
  keyframes: MotionKeyframeV1[];
}

interface MotionKeyframeV1 {
  frame: number;
  values: {
    x?: number;
    y?: number;
    rotation?: number;
    scaleX?: number;
    scaleY?: number;
    opacity?: number;
    renderSlot?: string;
  };
  easing?:
    | "linear"
    | "easeIn"
    | "easeOut"
    | "easeInOut"
    | { cubicBezier: [number, number, number, number] };
}

interface MotionEventV1 {
  frame: number;
  type: string;
  payload?: Record<string, string | number | boolean>;
}
```

约束：

- clip、track 和 event ID/类型均使用受限命名规则。
- FPS 为 `1..60` 整数。
- `durationFrames` 为正整数；合法帧为 `0..durationFrames`。
- 同一 clip 的 part track 唯一，同一 track 的 frame 唯一。
- 坐标建议限制为 `-10000..10000`，rotation 为 `-36000..36000`。
- authored scale 第一版限制为 `0.01..100`；镜像由 bind matrix 表达。
- opacity 限制为 `0..1`。
- renderSlot 必须由 rig 声明，并按离散属性采样。
- cubic bezier 的 x 控制点限制为 `0..1`，所有值必须有限。
- event payload 只允许一层标量 map，不允许脚本、函数或任意嵌套对象。
- `rigId` 必须与加载的 rig 完全相同。

`ProceduralChannel` 第一版冻结为项目当前实际需要的白名单，例如：

```ts
type ProceduralChannel =
  | "breathing"
  | "blinking"
  | "pointer-follow"
  | "hair-physics"
  | "ear-twitch";
```

新增通道必须版本化，不接受动作文件写任意 CSS selector 或 DOM 属性名。

## 9. 校验架构

校验分两层：

### 9.1 结构校验

JSON Schema Draft 2020-12 + Ajv：

- `additionalProperties: false`。
- Ajv strict mode 开启。
- 不启用类型强制、默认值注入或移除未知字段。
- schema 在模块初始化时编译一次并复用。
- 结构错误转换为稳定的 `ValidationIssue`，不直接向 UI 暴露 Ajv 内部对象。

### 9.2 语义校验

纯 TypeScript 检查：

- 重复 Part/clip/track/keyframe。
- logical parent 不存在或成环。
- sourceBinding 冲突。
- render slot 引用不存在。
- bind matrix 不可逆。
- track Part 不存在。
- motion rigId 不匹配。
- keyframe renderSlot 不存在。
- 事件不在允许的策略集合中。

公开 API：

```ts
type ValidationResult<T> =
  | { ok: true; value: T; warnings: ValidationIssue[] }
  | { ok: false; issues: ValidationIssue[] };

validateRig(input: unknown): ValidationResult<CharacterRigV1>;
validateMotionLibrary(
  input: unknown,
  rig: CharacterRigV1,
): ValidationResult<MotionLibraryV1>;
```

不得使用 `any`，不得抛出无法定位字段的笼统错误。issue 至少包含：
`code`、`path`、`message`、`severity`。

## 10. 时间与插值语义

### 10.1 帧时间

```text
continuousFrame = elapsedMs × fps / 1000
```

- 非循环 clip：钳制到 `0..durationFrames`。
- repeat clip：播放时对 `durationFrames` 取模；编辑器手动定位仍允许查看末帧。
- 采样接受小数帧，关键帧位置仍为整数。
- 纯函数不读取 `performance.now()` 或 `Date.now()`。

### 10.2 稀疏属性关键帧

每个属性独立寻找前后关键帧。某一帧没有写 `x`，不代表 `x=0`，而是该帧不参与
x 轨道：

```text
frame 0:  { x: 2, rotation: 0 }
frame 12: { rotation: -55 }
frame 24: { x: 4, rotation: 0 }
```

在 frame 12：

- x 在 frame 0 与 24 之间插值。
- rotation 在 0、12、24 之间分段插值。

规则：

- 属性没有任何关键帧：使用 authored identity 默认值。
- 首个属性关键帧之前：使用 identity 到首值的策略；P1 固定为首值保持。
- 末个属性关键帧之后：保持末值。
- easing 属于前一个“包含该属性”的关键帧到下一个属性关键帧的出段。
- rotation 按字面角度数值插值，不自动走最短路径，以允许未来表达多圈旋转。
- renderSlot 使用阶梯保持，不做数值缓动。

### 10.3 缓动

实现：linear、easeIn、easeOut、easeInOut、cubic bezier。

cubic bezier 使用 Newton-Raphson 并以二分法回退，必须测试：

- 端点严格为 0/1。
- 输出有限并在 `0..1`。
- 非法 x 控制点被 schema 拒绝。
- 极平切线时不发散。

## 11. 仿射矩阵与 Rig 求值

`Affine2D` 统一使用 SVG 2D 矩阵 `[a,b,c,d,e,f]` 和列向量约定。公开纯函数：

```ts
identity();
multiply(left, right);
invert(matrix);
translate(x, y);
rotateDegrees(angle);
scale(x, y);
transformPoint(matrix, point);
determinant(matrix);
approximatelyEqual(a, b, epsilon);
composeAroundPivot(transform, pivot);
```

要求：

- 不依赖 DOMMatrix，浏览器 adapter 负责 DOMMatrix 到 tuple 的转换。
- inverse 对奇异或近奇异矩阵返回结构化失败，不产生 Infinity/NaN。
- 统一 epsilon，不在不同函数中散落魔法数。
- 归一化 `-0` 为 `0`。

世界姿态：

```text
local(part) = bindMatrix(part)
            × authoredMotion(part)

world(part) = world(logicalParent)
            × local(part)
```

P1 只计算 authored 世界姿态；procedural 与 interaction 的最终叠加留给 P3，但接口
预留独立矩阵输入，不把它们烘焙到 motion 数据。

`renderSlot` 单独返回，不参与 world matrix。resolver 必须先做父级拓扑排序，输入
数组顺序不能改变结果。

## 12. 稳定序列化与指纹

`JSON.stringify()` 的当前插入顺序不算正式稳定序列化。P1 定义 canonicalizer：

- 对象键使用 schema 指定顺序，其余允许字段按字典序。
- parts 按 Part ID 排序。
- clips、tracks 按 ID 排序。
- keyframes 按 frame 排序，events 按 frame 后 type 排序。
- tags 和 suppress channel 排序去重。
- renderSlots 保持原顺序，因为它表达绘制顺序。
- 数值统一 `-0 -> 0`，禁止非有限数值。
- UTF-8、LF、2 空格缩进、文件末尾一个换行。

公开 API：

```ts
serializeRig(rig): string;
serializeMotionLibrary(library): string;
canonicalizeArtworkText(text): string;
sha256CanonicalText(text): Promise<string>;
```

相同语义对象无论输入对象键顺序如何，输出字节和指纹必须一致。

## 13. P0 数据迁移

提供一次性 `fromP0` 转换器，但不让生产运行时支持实验 schema：

1. 读取 `m8-p0@1`。
2. 要求 sourceFingerprint 与当前 canonical artwork 一致。

## 15. 自动化测试矩阵

### Schema / 校验

- 合法最小 rig、当前小洛宝 rig、第二套最小 rig。
- 未知 schemaVersion、未知字段、错误类型、非有限值。
- 重复 Part/slot/clip/track/frame。
- parent 缺失、self-parent、两节点环和多节点环。
- 未知 Part、未知 renderSlot、rigId 不匹配。
- 奇异 bind matrix、非法 viewBox、非法 scale/opacity/bezier。

### Matrix / Pivot

- identity、乘法顺序、inverse round-trip。
- translate/rotate/scale 和 pivot 组合。
- 镜像、斜切、非统一缩放和负坐标。
- logical parent 与 DOM 绘制顺序分离。
- 头部移动时马尾连接点跟随，但 renderSlot 保持 behind-body。
- 右手 renderSlot 切换不改变世界连接点。

### Timeline

- 空轨道、单关键帧、首尾和小数帧。
- 稀疏属性互不重置。
- 所有缓动端点和中点。
- repeat 边界和非循环钳制。
- rotation 字面插值、opacity、scale。
- renderSlot 阶梯保持。
- 输入 keyframes 未排序但函数不修改输入。

### Serialization / Migration

- 不同对象键顺序得到相同字节。
- 数组规范排序与 renderSlots 保序。
- `-0` 规范化，非有限值拒绝。
- CRLF/LF artwork 得到相同 canonical fingerprint。
- P0 wave 转换后采样帧 0/12/24 与预期一致。
- serialize -> parse -> validate -> serialize 字节完全一致。
