# M14 TTS 与口型验收报告

> 状态：代码与自动化检查已完成，等待真实 Windows/Tauri 验收。
>
> 范围：本机系统语音、语音设置、停止与打断、两态嘴型、回复自动朗读、Agent `pet_say` 和音乐反应协调。不包含外部 TTS 服务、录音、语音识别或语音音频缓存。

## 1. 已实现边界

- 默认关闭。启用后只使用 WebView2 暴露的 Windows 本地系统音色，不新增 API key、网络 endpoint 或应用音频缓存。
- `speech.say` 经 `SpeechController`、独立 `speech` 调度通道和统一执行器运行；新的高优先级或 `interrupt=true` 语音可以打断旧语音。
- 设置窗口支持音色、语速、音高、试听、停止以及“自动朗读模型回复”。关闭全局声音或将音量设为 0 会立即停止朗读。
- Agent 只在设置开启、全局声音可用且当前 WebView 支持系统语音时看到 `pet_say`。工具文本限制为 500 个 Unicode 字符，仍经过动作校验、频率策略与调度器。
- 角色使用 `mouth_closed` 与 `mouth_open` 两层。Web Speech 不提供音素或音量样本，因此首版用确定性本地包络生成开口量；未来 Provider 可以通过 `onMouthLevel` 提供更精确参数。
- `mouth_open` 按最大张嘴素材处理；当前 Web Speech 普通朗读经非线性映射后限制在 `8%～55%`，横向与纵向同步缩放，并使用 `90ms` 补间。像素两态嘴默认关闭透明度交叉淡入，只对尺寸做补间；需要时可在 `speechMouth.crossFadeEnabled` 重新开启。更大的重读幅度保留给未来 viseme/音频包络 Provider。
- `prefers-reduced-motion` 下不循环快速张合，而保持克制的静态开口。说话期间只降低角色自身的音乐律动，不操作系统播放器或其他应用音量。

## 2. 人工验收步骤

1. 运行 `npm run tauri dev`，右键小洛宝打开设置，确认“本机语音与口型”默认关闭。
2. 保持全局声音开启，启用语音朗读，选择一个中文本地音色，调整语速和音高后点击“试听”。
3. 预期：小洛宝发声；闭嘴/开嘴图层交替且位置稳定；朗读结束后恢复闭嘴，不影响眨眼、视线、马尾或点击招手。
4. 在朗读中点击“停止朗读”。预期：声音立即停止，嘴型归零；随后再次试听仍能正常工作。
5. 连续快速点击两次试听。预期：后一次打断前一次，不重叠播放，不留下持续张嘴状态。
6. 开启“自动朗读模型回复”，在对话窗口发送一条消息。预期：模型完整回复成功后朗读一次；停止生成、请求失败或空回复不朗读。
7. 开启 Agent，让模型“用一句简短的话向我问好，并使用语音说出来”。预期：工具日志显示 `pet_say` 完成，文本只朗读一次；如果模型本轮已调用 `pet_say`，不会再因自动朗读重复整段回复。
8. 播放系统音乐并开始朗读。预期：角色自身的音乐律动暂时减弱，系统音乐不暂停、不降音量，朗读结束后角色律动恢复。
9. 关闭全局声音，或关闭语音设置。预期：正在播放的语音立即停止，Agent 下一轮不再获得 `pet_say`。
10. 在 Windows“减少动态效果”偏好下重复试听。预期：口型采用静态克制开口，不做快速循环。

## 3. 降级与隐私检查

- 若系统语音 API 不存在，角色和对话仍可正常使用，Agent 不暴露 `pet_say`；普通 Agent turn 不因语音不可用而中断。
- 应用不保存合成音频，`SpeechController.getCacheStatus()` 恒为 `{ entries: 0, bytes: 0, policy: "none" }`。
- 本阶段没有录音、麦克风权限、语音识别、外部 TTS 请求或新增 Tauri capability。

## 4. 自动化检查

- `WebSpeechController`：默认关闭、本地音色过滤、朗读生命周期、口型归零、停止与零缓存策略。
- `PetActionExecutor`：`speech.say` 分发、连续口型和结束清理。
- Agent 工具：运行时能力过滤、严格参数 schema 和 `speech.say` 映射。
- 设置：schema v7 默认值、旧版本迁移与语速/音高范围。
- 口型包络：确定性、有限范围与非法时间降级。
