## 4. Schema 与共享核心

- 使用 Ajv 8 的 Draft 2020-12 validator，`strict: true`、`validateSchema: true`。
- rig 校验命名、正尺寸 viewBox、唯一 Part/binding/slot、父级无环和可逆 bindMatrix。
- motion 校验 FPS、合法帧、数值范围、唯一 track/keyframe、Part/slot 引用、事件和
  procedural channel 白名单。
- 稀疏关键帧按属性独立采样；缺省 easing 为 linear；离散 renderSlot 使用 step hold。
- canonical serializer 保持 renderSlots 顺序，稳定排序其余集合，归一化 `-0`，对
  NaN/Infinity 抛错而不是改写为 0。
- `dist/index.js` 已通过真实 Node ESM import 验证，不再依赖 Vite alias 才能使用。
- 核心不依赖 React、DOM、svgcanvas、Tauri、矩阵库或缓动库。

## 5. 编辑器闭环

编辑器现在明确区分：

1. 初始化 svgcanvas。
2. 载入角色样例并建立真实 rig。
3. 显式载入挥手动作示例，或导入与当前 rig 匹配的 motions v1。

导入动作按钮在角色未载入时禁用。导入使用 `validateMotionLibrary()`，会检查 rigId、
Part 和完整 schema；成功后自动选择第一条可播放轨道并回到第 0 帧。内存中保留完整
动作库，不丢失多 clip、多 track、FPS、时长、循环、缓动、事件、renderSlot 或
procedural channel，导出使用同一个 canonical serializer。

默认挥手不再伪装成 SVG 自带动作，而由独立“载入挥手示例”按钮创建。
