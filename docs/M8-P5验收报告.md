## 2. 已实现

### 2.1 兼容、备份与恢复

- 前端与 Rust 同时声明 project、rig、motions 的当前及支持 schema 范围；未知或更高版本在打开、保存和发布前被明确拒绝。
- 覆盖保存前把旧项目写入 `.ltypet-backups/`，最多保留 5 份；恢复旧备份前再次备份当前项目，避免恢复动作本身造成不可逆覆盖。
- 正常恢复入口为：打开原项目 → 右侧“诊断” → 展开“项目备份（n/5）” → 点击对应时间的“恢复”。不要求也不建议在文件选择器中直接打开内部备份目录。
- 保存继续使用 staging、journal 和原子替换；半写事务在重新打开时回滚。
- Rust 测试覆盖保存前备份、恢复及未知 schema 不得写入。
- 0.1.0 之前没有公开项目 schema，因此本版不伪造旧版迁移器；后续首次引入新 schema 时必须先迁移副本，再让用户确认保存。

### 2.2 诊断与隐私

- 增加本地“导出诊断”，只包含程序版本、OS/架构、schema 支持范围和 recovery/最近项目数量。
- 诊断不包含项目路径、SVG、动作内容、用户文本或密钥。
- 正式编辑器不提供 LLM、遥测、网络上传、任意 shell 或通用文件系统能力；开发专用的固定生产发布入口在正式构建中关闭。

### 2.3 上游与供应链

- 根应用、`packages/character-motion` 和 `tools/motion-editor` 的 lockfile 均纳入版本控制，干净构建使用 `npm ci`。
- `@svgedit/svgcanvas` 固定为 `7.4.2`，记录 npm tarball 地址和 SRI。上游 npm 包未提供可验证的 `gitHead`，文档没有虚构 commit；当前没有修改 `node_modules` 或维护私有 svgcanvas patch。
- 生成 `THIRD_PARTY_LICENSES.generated.json`：204 个 npm 包、240 个 Windows Rust 包，许可证识别结果 0 个 `UNKNOWN`。
- 增加 [`UPSTREAM.md`](../tools/motion-editor/UPSTREAM.md)、[`THIRD_PARTY_NOTICES.md`](../tools/motion-editor/THIRD_PARTY_NOTICES.md)、[`CHANGELOG.md`](../tools/motion-editor/CHANGELOG.md) 和 [`RELEASE.md`](../tools/motion-editor/RELEASE.md)。

## 3. Release 黑屏问题与修复

开发态界面正常，但首次启动 release 程序时页面未渲染。加入只在启动失败时显示的前端错误边界后，定位到严格 CSP 拒绝 `unsafe-eval`：共享动画包在模块加载阶段让 Ajv 动态编译 schema，生产 WebView 因而在 React 挂载前失败。

修复方式不是放宽 CSP，而是：

1. 在 `packages/character-motion` 构建前用 Ajv standalone 生成 ESM 校验器。
2. 运行时直接导入预生成的 rig/motions 校验函数，不再执行动态代码生成。
3. `release:check` 额外拒绝编辑器产物中的 `new Function` 和 `eval(`。
4. 保留启动错误界面，使未来 release-only 初始化错误可以直接诊断。

修复后，严格 CSP 下的正式便携程序已实际启动，成功显示界面并载入正式小洛宝项目：33 个 Part、33 个 pivot、3 个 Clips。重复启动验证只保留一个窗口。

## 6. 最终发布门结果

以下项目由用户在后续人工验收中完成并确认没有其他问题：

- 在没有 Node、Rust、仓库源码和旧 `dist` 的 Windows 10/11 x64 环境安装或解压启动。
- 打开当前小洛宝项目，编辑、保存、关闭重开、恢复异常会话并导出 canonical 资产（内置历史备份恢复按钮另见下文）。
- 覆盖安装升级、版本降级、卸载后确认外部项目、`.ltypet-backups` 和 AppData recovery 不被误删。
- 100%、150%、200% DPI，多显示器和负坐标，休眠恢复，以及不同 WebView2/显卡环境。
- 长时间轴、密集关键帧、较大 SVG 和长时间反复播放/撤销的内存、监听器、RAF、对象 URL 与 DOM wrapper 稳定性。
- 安装与卸载后的项目、备份数据保留正常。

备份文件已经确认存在且可以人工读取。正式恢复按钮位于已打开项目的右侧“诊断”面板中；直接选择 `.ltypet-backups` 内部目录不是正式交互路径。该说明已补入发布文档。P5 封板前只需在测试项目中点击一次内置“恢复”，确认动作回到旧状态，且恢复前版本被再次保留。

公开分发前对角色素材、图标的最终权利审核，以及安装包签名策略仍属于实际发行操作，不阻塞 M8 P5 工程阶段封板。
