# M11 对话壳与模型供应商层验收报告

> 实施日期：2026-07-22  
> 范围：独立文本对话、Provider 抽象、流式取消、上下文预算和系统安全密钥存储；不包含 Agent 工具调用、长期记忆与 TTS。

## 1. 结论

M11 已形成“对话 UI → Provider 契约 → 原生模型通信”的独立闭环。默认离线 Mock 不联网，外部 OpenAI-compatible Provider 只有在用户明确同意外发后才能发送；模型没有角色、窗口、文件或原生命令工具。

## 2. 实现边界

- `chat.html` 是独立 Tauri 窗口，可从角色原生菜单、托盘和设置页打开。
- Enter 发送、Shift+Enter 换行、Escape 或按钮停止；加载、部分输出、错误和恢复输入状态均可见。
- 会话只保存在聊天窗口内存，关闭后不持久化，也不属于长期记忆。
- 外发内容仅包含当前输入及字符预算范围内的最近 user/assistant 消息；不附带屏幕、窗口、前台应用、番茄钟或其他感知数据。
- Provider 契约位于 `src/domain/chat/`，离线 Mock 与 OpenAI-compatible 适配位于 `src/providers/`，均不依赖角色控制器。
- OpenAI-compatible 使用完整 Chat Completions 地址和模型名；原生层强制 HTTPS，仅允许 localhost/127.0.0.1/::1 使用 HTTP。
- 请求超时限制为 3-120 秒；首个增量前最多自动重试 2 次；收到部分输出后不重试，避免重复文本。
- 上下文预算使用 Unicode 字符数做供应商无关的保守限制，并优先保留最近消息；原生层另有 100 条消息与 100000 字符硬上限。

## 3. 密钥与隐私

- API key 不进入 `settings.json`、源码、仓库或日志，也不通过读取命令返回 WebView。
- 前端只能查询“是否存在”、覆盖或删除；Rust 发起请求时从 Windows DPAPI 存储解密单个供应商密钥。
- `secrets.dpapi` 默认绑定当前 Windows 用户和机器；系统管理员强制重置该用户密码等 DPAPI 限制可能导致旧密钥无法恢复，此时重新保存即可。
- 若应用数据目录存在旧版 `secrets.json`，首次启动会在成功加密后删除明文；迁移失败保留原文件并只记录不含密钥内容的错误。
- 外部 Provider 默认未启用、外发同意默认关闭；设置页明确显示外发范围。
