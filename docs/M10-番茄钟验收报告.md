# M10 番茄钟验收报告

> 实施日期：2026-07-22  
> 范围：M10「番茄钟与第一个本地业务能力」

## 1. 结论

M10 已形成完整纵向链路：设置页与动作控制台通过统一 `TauriTimerController` 调用 Rust 计时状态机；计时完成后由运行时向 `BehaviorScheduler` 提交语义动作，并由原生层按设置提供用户注意提醒和提示音。

计时不依赖 WebView 的 interval 累加。前端的 500ms interval 只刷新倒计时文字，删除、卡顿或暂停该刷新都不会改变 Rust 中的真实截止时间。

## 2. 实现边界

- 同一时间只允许一个运行中或暂停的计时器；重复开始返回 `timer_conflict`。
- 支持 `focus`、`break`、`custom` 三类计时及开始、暂停、继续、取消。
- 运行中状态保存绝对 Unix 截止时间；暂停状态保存冻结的剩余毫秒数。
- 状态保存在应用数据目录的 `timer-state.json`，写入失败会回滚内存状态；旧文件保留为 `.bak`。
- 应用重启后恢复运行中或暂停的计时。若重启时已经到期，后台检查会立即完成。
- 休眠或系统时钟向前跳变后，按当前墙上时钟校正并只完成一次。
- 系统时钟向后调整时遵循绝对截止时间：计时延后到原截止时间再次到达，不把倒退区间算作已过去时间。
- 完成快照先持久化，再发出 `timer-finished-available`；主窗口领取后清除，避免重启或多窗口造成重复角色提醒。
- 完成表现是 `source: "timer"`、`priority: "timer"` 的 `motion.play`，不会直接操作 React 角色组件。
- 声音开关同时受番茄钟 `soundEnabled` 与全局 `audio.enabled` 约束；系统提醒与声音均在计时开始时写入该计时快照。

## 3. 设置与入口

设置 schema 已从 v1 升级到 v2，新增：

```json
{
  "pomodoro": {
    "focusMinutes": 25,
    "breakMinutes": 5,
    "showSystemReminder": true,
    "soundEnabled": true
  }
}
```

- 设置窗口：配置默认时长和提醒偏好，显示当前计时，提供开始专注、开始休息、暂停、继续、取消。
- 动作控制台 / Mock Agent：支持 `timer.start`、`timer.pause`、`timer.resume`、`timer.cancel`，并展示 timer capability。
- 跨窗口保存设置后发出 `settings-changed`，避免主窗口以旧内存快照覆盖设置窗口刚保存的番茄钟偏好。

## 5. 真实窗口验收步骤

已观察：用户于 2026-07-22 自行运行当前开发版本，确认小洛宝主窗口可以正常持续显示，未复现“闪现后消失”。番茄钟交互、完成提醒与重启/休眠恢复仍按以下步骤验收，未观察的项目不记为通过。

1. 运行 `npm run tauri dev`，从托盘或右键菜单打开“设置”。
2. 在“番茄钟”中把专注/休息时长改为合适值，关闭再打开设置窗口，确认值仍然存在。
3. 点击“开始专注”，确认状态每秒递减；暂停等待几秒，数字应保持；继续后再次递减；取消后回到“当前没有计时”。
4. 在动作控制台提交短时 `timer.start`，记录自动回填的 timer ID，再用 `timer.pause`、`timer.resume`、`timer.cancel` 验证 Mock Agent 路径。
5. 让短时计时自然结束，确认只招手一次；按设置确认系统注意提醒与提示音开启或关闭。
6. 启动计时后退出并重启应用，确认计时按绝对截止时间恢复；暂停后退出重启，确认暂停剩余时间不变。
7. 在运行中让系统睡眠至超过截止时间，唤醒后确认立即完成且不重复提醒。
