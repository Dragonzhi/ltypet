## 1. 信任边界

外部状态不能直接成为可信动作。`ObservationHost` 依次执行：

1. 校验协议版本、信封字段和事件类型。
2. 限制 payload 为 4 KiB，并按事件类型拒绝任何未声明字段。
3. 检查来源 kind/ID、逐来源 grant 和最高敏感级别。
4. 检查事件时效、未来偏差、重复 ID、滚动频率预算和本地安静时段。
5. 使用本地固定规则生成反应意图，并重新经过 M1 Action 与当前能力校验。
6. 以显式 `agent` 优先级提交 `BehaviorScheduler`，不抢占用户、菜单或计时提醒。

诊断只保留事件 ID、来源、类型、结果、错误码和动作类型，不保留 payload。默认不把观察事件送入模型，也不存在外发路径。

## 4. 开发态人工冒烟

1. 运行 `npm run tauri dev`。
2. 在设置中开启“允许外部状态触发角色反应”。
3. 展开开发态动作控制台的“M13 观察事件”。
4. “等待用户”应触发短暂眨眼；“任务完成”应招手；“任务失败”应播放 `bow`。
5. “媒体播放（当前无动作）”应返回 `ignored`，角色不播放伪造的音乐动作。
6. 点击“立即停止所有自主行为”后，新测试事件应返回 `paused`；关闭再开启外部反馈总开关后恢复。
7. 开启安静时段并把范围设为覆盖当前时间，新测试事件应返回 `quiet_hours`；关闭安静时段后恢复。
