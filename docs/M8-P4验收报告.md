真实验收曾发现 `wave` 期间视线跟随停顿，结束后跟随基准偏移。根因是 `SvgRuntimeRig` 每帧投影全部 33 个 Part，并使用包含实时 interaction transform 的当前父级 CTM 求逆，导致未参与动作的跟随层被冻结，当前鼠标位移又在恢复时被烘入 authored wrapper。现已改为只投影 Clip 控制的 Part 及逻辑后代，使用初始化时稳定的父级 bind CTM 组合 authored 动作，恢复时直接写回 bind transform。DOM 回归测试覆盖外层跟随在播放和恢复期间持续生效、未参与 `wave` 的 Part 不被运行时改写。

后续真实验收又发现 `wave` 开始/结束时领带和双臂待机微动突变，以及双臂默认高度偏移。原因是 `wave` 曾暂停包含根呼吸、领带和双臂微动的整个 `breathing` 通道，并且 runtime 在恢复 CSS 程序动画后才测量 bind 对齐，使初始化时的动画相位可能被烘入 authored wrapper。现已让 `wave` 不抑制任何程序通道，其他动作的 CSS 通道暂停使用 `animation-play-state: paused` 保持当前相位；runtime 在所有程序/interaction 层处于中性状态时建立 bind 对齐，再恢复程序动画。实际桌面复测仍需确认领带/手臂无突变且肩部连接恢复。

本轮测试进程已在汇报后停止。后续用户已完成真实窗口验收并确认进入 P5。当前状态为：**P4-0 至 P4-5 已完成，P4 已封板。**
